What the Live Map Shows

Every element on your admin map is real-time, updated every 2–5 seconds via Socket.IO.

Driver Positions

Every active driver broadcasts GPS coordinates every 2–5 seconds. Available drivers appear in green; on-ride drivers in amber.

Live Ride Routes

Active ride routes are rendered in real time using Mapbox GL, showing the optimized path from pickup to dropoff with traffic awareness.

SOS Pins

Emergency SOS triggers appear as red pins instantly. Admins can see the driver's exact location and resolve incidents from the dashboard.

Pickup & Dropoff

Rider-requested pickup and drop locations are pinned on the map and linked to the nearest available driver automatically.

Demand Heatmap

Overlay a demand heatmap derived from historical ride data to predict where riders will request next — helping pre-position drivers.

Manual Override

Admins can manually assign any available driver to any open ride request directly from the map, bypassing automatic AI matching if needed.

How the Live Map Works

1. Driver App Broadcasts

The driver mobile app sends GPS coordinates to the backend via Socket.IO every 2–5 seconds during an active session.

2. Redis Caches Position

Each coordinate update is cached in Redis with a geospatial index for sub-millisecond proximity lookups during dispatch scoring.

3. Admin Map Re-renders

The admin dashboard receives the position event and smoothly updates the Mapbox GL layer — no page refresh required.
